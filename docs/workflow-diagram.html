01 · Complete Workflow

From Lead to Payment — the full process

Each stage is a board / department. Every node below shows which department owns it, which automation fires, and which tools are involved.

01

Facebook Ads

The lead source. Ads run, and an interested candidate is driven to the intake form.

EXTERNAL · MARKETING Facebook Lead Ads
🔗 Integration: Facebook Lead Ads flow directly into the Candidate Form / board.
02

Candidate Form

The candidate submits their details (name, email, phone, program). This is the entry point of intake.

INTAKE Form Builder ＋ De-dup by email
⚡ Automation: if the email already exists → the existing item is updated instead of creating a duplicate.
03

Sales Board

The new lead lands in the “New Leads” group (blue). The sales process starts here.

SALES Groups · Status · Person
⚡ Automation: item created → group = New Leads, status = New.
04

Auto-Assign Sales Person Round Robin

The lead is automatically rotated and assigned to a sales consultant — fair distribution.

SALES Person Column ＋ Built-in Round Robin
⚡ Automation: new lead → assign to next consultant → notify them.
05

Consultation

The consultant speaks with the candidate. Status becomes “Contacting” (yellow); the footprint is saved in updates/bubble.

SALES Updates / @mention Status → Contacting
⚡ Automation: status change → item moves to that group + the department is tagged.
06

Visa Process

A qualified candidate enters the visa stage. Visa expiry dates are tracked and reminders are sent.

VISA Date Columns Reminders
⚡ Automation: notify 1mo / 3wk / 2wk / 1wk / 5d / 3d / 2d / 1d before visa expiry + email.
07

Contract Generation

A contract / job-offer document is generated from board data — using a template + placeholders (300+ employer templates).

DOCUMENTS DocuGen Template + Placeholders → PDF/Word
⚡ Automation: status = Generate → build document → save to File column (“output”).
08

DocuSign

The candidate signs the contract. The signature is pulled into the board via a mirror column.

DOCUMENTS DocuSign · E-signature Mirror Column
⚡ Automation: signed → status update → trigger the next stage (Finance).
09

Finance / Accounts Build FIRST

Every department's invoice request arrives here. Accounts verifies it — only then is an invoice created.

ACCOUNTS Invoice Intake → Verify ＋ Approval Stages
⚡ Automation: request → intake stage → Accounts approve → trigger invoice generation. Every department connects here.
10

Stripe Payment

Once verified, a Stripe invoice is auto-generated (amount, territory, email pre-filled) and emailed to the candidate.

ACCOUNTS Stripe · PTY (offshore) Stripe · Global (Dubai) ＋ Offline / Serop capture
⚡ Automation: approved → Stripe invoice → auto-email → paid status updated on the board.
11

Dashboard Report

Data from all boards flows into one cross-department dashboard — monthly reports, charts, KPIs. Click any number to verify.

REPORTING Cross-board Charts ~1 min auto-refresh
📊 Only “Done” items are counted — mistriggered/forgotten statuses are caught via drill-down.
✓

Completed

Candidate placed, payment received, records in the dashboard. The group turns green — the process is complete.

02 · monday.com vs Client

What's the same, what's extra

monday.com is a general project-management tool. The client wants their entire business system on a similar UI.

Baseline · already exists

What monday.com offers

  • ✓Users, roles (Admin / Member / Viewer)
  • ✓Workspaces, boards, groups, columns
  • ✓Status / Date / File columns
  • ✓Forms → responses into the board
  • ✓Automations (limited)
  • ✓Connected boards + mirror columns
  • ✓Dashboards, charts, filters & views
  • ✓Google login, invite users, @mentions
  • ✓Docs & e-sign — only via 3rd-party apps (DocuGen, DocuSign)
Custom · to be built

What the client wants

  • ＋All baseline features + custom roles + Developer
  • ＋Advanced automation builder — unlimited workflow engine
  • ＋A folder + search system for automations
  • ＋Duplicate prevention (email match)
  • ＋Built-in Round Robin, Document Generator, E-signature
  • ＋Full Finance module — Invoicing + Stripe workflow + accounting boards
  • ＋Candidate Tracking, Visa Tracking, HR Workflow
  • ＋Employer management + candidate→employer tagging
  • ＋Recruitment CRM — all in one app

03 · The 4 Improvements

Where we go beyond monday.com

Everything else is replicated. Only these 4 things are missing or limited in monday.com — these are the ones to improve.

GAP 01

Duplicate prevention on forms

monday: every form submission creates a new duplicate entry.

Client: match by email and update — no duplicate created.

GAP 02

Clickable links in emails

monday: automated emails can only send a plain URL.

Client: a clickable button / hyperlink that redirects the candidate.

GAP 03

Automation organization

monday: no folders or search for automations — it gets confusing.

Client: folders + search to manage many automations.

GAP 04

Candidate → Employer tagging

monday: not a native feature — currently done via a manual hack.

Client: easy native tagging + live counts (active / interview / placed).
